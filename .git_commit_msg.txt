fix(windows): auto-detect timezone + reduce SmartScreen friction

Dos quejas reportadas en el .exe de Windows.

────────────────────────────────────────────────────────
1) "no se detecta zona horaria automáticamente"
────────────────────────────────────────────────────────
Causa: en Windows el registro guarda nombres locales del SO
("China Standard Time", "Pacific Standard Time"…) en vez de
nombres IANA ("Asia/Shanghai", "America/Los_Angeles"). Las 3
estrategias previas de detect_system_timezone() las rechazaba
todas y la app caía a UTC.

Fix:
* pyproject.toml — nueva dependencia `tzlocal>=5.0` (cruza
  registro Windows → IANA usando la tabla oficial Unicode CLDR;
  en macOS/Linux también funciona y nos da fallback uniforme).
* timezone_service.detect_system_timezone() — nueva estrategia
  #3 que llama `tzlocal.get_localzone()` y devuelve `.key`. Las
  demás estrategias se mantienen como fallbacks.
* packaging/shakevision.spec — hidden imports añaden tzlocal,
  tzlocal.utils, tzlocal.windows_tz para que PyInstaller los
  meta en el bundle (sin esto la rama tzlocal silenciosamente
  fallaría en el .exe).

────────────────────────────────────────────────────────
2) "SmartScreen avisa de software no reconocido al abrir"
────────────────────────────────────────────────────────
Causa raíz: el .exe no está firmado con un certificado EV de
code signing. SmartScreen bloquea TODO PE sin firma bajada de
internet — es comportamiento por diseño de Windows.

No se puede eliminar el warning sin un certificado ($300–500/año,
en la roadmap v1.0). Lo que SÍ se puede hacer ahora es reducir
la fricción de varios pasos:

* packaging/windows/version_info.txt — nuevo recurso
  VS_VERSIONINFO con CompanyName, ProductName, FileDescription,
  versión 0.1.1.0. Sin esto el PE es totalmente anónimo y
  SmartScreen muestra su política más estricta (banner rojo
  "no se puede ejecutar"). Con la tabla bien formada baja al
  diálogo azul con botón "Más información → Ejecutar de todas
  formas" — mucho menos hostil.
* packaging/shakevision.spec — EXE() ahora pasa
  version=WIN_VERSION_INFO (solo aplica en Windows; otras
  plataformas lo ignoran).
* README.md — sección nueva "🛡 Windows SmartScreen / macOS
  Gatekeeper" con instrucciones explícitas para Windows
  ("Más información → Ejecutar de todas formas") y macOS
  (right-click → Open la primera vez).
* Roadmap v1.0 actualizado: code signing EV + Apple Developer
  ID + notarización.

CHANGELOG.md actualizado con todos estos puntos en la sección
[0.1.1].
