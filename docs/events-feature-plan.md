# 事件功能区分规划(v0.7.7)

两处事件入口,职责清晰分工,**共用同一套派发逻辑**(MainWindow
`_open_event_in_replay`),不重复造轮子。

## 一、顶栏「事件」中心(Standard,与地球/数据同级)
**定位:发现 + 选事件 + 选就近台站 + 派发回放。** 完整工作流入口。

- 左:USGS 事件表(可排序:时间/震级/深度/地点)。
- 右:选中事件 → **最近台站**列表(台站码 / 震中距 Δ° / 位置),
  本地用 `great_circle_degrees` 算,**零联网**(台站坐标已常驻)。
- 双击某台站 → `review_requested(quake_id, station)` → 打开工作台回放,
  **用该台站**复核(就近 ⇒ 窗口合理 ⇒ P/S 落在数据里)。

## 一b、顶栏「我的」中心(v0.7.7,替代工作台「本地」)
决定:把"本地"升级为顶栏「我的」(与地球/数据/事件并列)。两块:
- **收藏**:★地震(双击复核,FavoriteEvent 现存坐标)+ ★台站(双击使用)。
- **记录**:触发录制(**C:有才显示**)+ 复核目录(QuakeML)。
每表带动作(取消收藏/删除/打开)。工作台降为 4 子标签(纯分析)。

## 二、工作台「事件」tab —— 已移除(v0.7.7 UX)
经讨论删除:顶栏事件中心已覆盖且更强,避免双窗口重复。工作台聚焦分析,
子标签降为 5 个(实时/24h/质点/回放/本地)。若以后分析中频繁换事件、切窗烦,
再在回放面板内加轻量「其他事件 ▾」重选(不再做完整重复表)。

## 三、事件 → 回放 的机制(关键:与"加台站/连接"解耦)
- 回放走 **IRIS dataselect 档案(HTTP)**,**不连接 SeedLink、不碰侧栏台站
  下拉**。所以复核事件**不需要"添加台站再连接"**,也不会和实时连接冲突。
- `ReplayPanel.set_event_review(...)` **直接**设回放的 N.S.L.C.(就近台站),
  绕开组合框选择逻辑(那条是给实时连接用的)→ 彻底避免重复。
- 加载后:有台站坐标 → TauP 叠加理论 P/S,状态显示**震中距 Δ**;
  顶部 **事件横幅**显示"正在复核:M6.2 — 地点 · 台站 IU.XXX"。

## 四、台站/事件距离的现实
IU/US 全球约 150 台,稀疏。"最近台站"未必很近(可能仍是区域/远震),
但远胜于"碰巧选中的台站"。Δ 显示帮用户判断这次复核的远近与可用性;
窗口没覆盖 P 时提示加大时长(已实现)。
