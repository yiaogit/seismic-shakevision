# 工作台(Workbench)专业度评估与改进方案

> 视角:一名在台网/台站做事件分析的地震分析员实际使用本软件。
> 基线:v0.7.7(实时管线已修好,IRIS 三分量波形连续、Hodograma 可用)。

---

## 1. 当前工作台具备什么(做得好的)

| 能力 | 现状 | 评价 |
|---|---|---|
| 实时波形(3 分量) | SeedLink,重采样到统一率,hold 填充 | ✅ 现在连续干净 |
| 频谱图 | 滑窗 FFT,幅度 dB | ✅ 够用 |
| Helicorder(Diario 24h) | 24h 滚筒,min/max 包络,99 分位归一 | ✅ 专业做法 |
| 质点运动(Hodograma) | N-E 平面,色带拖尾 | ✅ 可看极化 |
| 带通滤波 | Butterworth,低/高/阶可调 | ✅ 但全局、非交互 |
| STA/LTA 触发 | 阈值/STA/LTA 可调 + 迟滞 | ✅ |
| 事件录制 | 触发存 MiniSEED | ✅ |
| 历史回放 | IRIS dataselect 下载 + 倍速 + 游标 | ✅ |
| 声化 | 加速重放为音频 | ✅ 特色 |
| 连接可视化 | spinner + 实时状态行(v0.7.7) | ✅ 新增 |
| 呈现层 | 主题/4 语言/3D 地球/强度卡 | ✅ 打磨度高 |

**定位结论**:这是一台**打磨精良的"单台站实时监测 + 可视化"工具**,
教育/发烧友到准专业层级,呈现力强(对标 Raspberry Shake 的 Shake、
SWARM 的实时视图、jAmaSeis)。

## 2. 从地震分析员视角的差距(专业度短板)

一名分析员处理一次地震的典型动作:多台站对比 → **拾取 P/S 震相** →
量 **S-P 走时**估距、量**振幅**估震级 → **去仪器响应**换成物理单位 →
交互试不同滤波 → 自由缩放/选窗 → 谱/PSD 分析 → 极化/反方位角 → 定位 →
进事件目录、导出震相(QuakeML)。

对照之下,当前工作台缺的核心分析环节:

| 能力 | 现状 | 分析员需求 |
|---|---|---|
| **震相拾取(P/S)** | ❌ 无 | ⭐ 必需 |
| **测量游标(时间/振幅、S-P)** | ❌ 无 | ⭐ 必需 |
| **去仪器响应 / 物理单位** | ❌ 显示的是原始 counts(±5000) | ⭐ 必需 |
| **震级估计** | ❌ 只显示 USGS 目录震级,不能从本地波形量 | ⭐ 重要 |
| **冻结/缩放/选窗分析** | 部分(实时只有水平缩放) | ⭐ 重要 |
| **多台站对比** | ❌ 一次只一个台站 | ➕ 重要 |
| **PSD / 噪声谱(PPSD)** | ❌(只有幅度谱图) | ➕ |
| **反方位角(从质点运动)** | ❌ 只可视化、无读数 | ➕ |
| **事件目录浏览/导出** | ❌(只存盘) | ➕ |

**最关键的三块**:
1. **去仪器响应 → 物理单位**。现在波形是 counts,跨仪器不可比;强度卡的
   PGV 用一个固定的 `gain_cm_s_per_count` 粗估(`default_gain_for`),
   对真正的台站是近似值。分析员离不开 velocity(m/s)/位移。
2. **交互拾取 + 测量**。没有点选 P/S、没有读时间/振幅、没有 S-P。
3. **冻结/选窗分析**。实时滚动之外,需要能停下来翻看、框选一段做测量。

**坦白结论**:**它目前是"监测仪表盘",还不是"分析工作台"。**
"Banco de trabajo / Pro" 的命名给的期待,比当前分析工具集要高。

## 3. 基于 PySide / pyqtgraph 的改进方案

好消息:现在用的 **pyqtgraph** 自带大量分析交互能力,目前基本没用上。
很多"专业感"的功能成本其实不高。

### A. 快速见效(pyqtgraph 原生,低成本高价值)
1. **十字游标 + 读数**:`SignalProxy` 接 `sigMouseMoved` + 两条
   `InfiniteLine` → 实时显示当前 时间 / 振幅。
2. **点选拾取 P/S**:`scene().sigMouseClicked` → 放可拖动的竖
   `InfiniteLine`(带 `label`)当 P/S 震相;自动算 **S-P** 并换算粗略
   震中距(S-P × 8 km/s 经验)。
3. **框选分析**:`pg.LinearRegionItem` 选一段时间窗 → 对该窗测峰值振幅、
   做 FFT/PSD、单独声化、或导出。
4. **冻结模式**:一个"暂停"按钮停掉滚动,放开 pyqtgraph 的平移/缩放,
   在已有的 5 分钟缓冲上做上面这些分析。

### B. 中等(scipy/已有依赖)
5. **逐通道滤波 + 预设**:每个分量独立滤波;给常用频带预设
   (体波 1-10Hz、面波 0.02-0.1Hz、微震 0.1-1Hz)。
6. **PSD / 频谱**:对选窗做 Welch PSD(`scipy.signal.welch`),显示主频。
7. **本地震级估计**:从选窗峰值振幅 + 距离(来自 S-P 或地球点击)算 ML。

### C. 重头(ObsPy / 架构,高价值高成本)
8. **去仪器响应**:ObsPy(已是依赖)从 IRIS FDSN station 服务取
   StationXML,`remove_response` 换成 velocity/displacement;UI 加
   counts ↔ m/s 切换。**专业度提升最大的一项。**
9. **多台站对比**:`GraphicsLayoutWidget` 纵向堆叠 N 台站(或分页)。
10. **事件目录浏览器**:把触发录制的事件做成可浏览列表 + 元数据,导出
    QuakeML/CSV。

## 4. 建议优先级(分析员价值 ÷ 成本)

| 优先 | 项 | 价值 | 成本 | 风险 |
|---|---|---|---|---|
| P0 | 十字游标读数 + 框选测量 + 冻结模式 | 高 | 低 | 低(纯 pyqtgraph,不碰数据路径) |
| P0 | 点选 P/S 拾取 + S-P 读数 | 高 | 中 | 低 |
| P1 | 去仪器响应 → 物理单位 | 很高 | 高 | 中(ObsPy/StationXML,需联网+缓存) |
| P1 | 逐通道滤波 + 频带预设 | 中 | 低 | 低 |
| P2 | PSD/主频、本地震级估计 | 中 | 中 | 低 |
| P2 | 多台站对比 | 中 | 高 | 中 |
| P3 | 事件目录浏览 + QuakeML 导出 | 中 | 中 | 低 |

**建议路线**:先做一批 **P0 的 pyqtgraph 交互**(游标/拾取/选窗/冻结)——
它们最能立刻把"监测仪表盘"变成"分析工作台",而且全在 UI 层、不碰刚修好的
实时数据管线、风险低、沙箱里也能写。再视需要推进 P1 的去响应(那是真正
让数据"专业可用"的一步,但要联网取 StationXML,值得单独立项)。

> 说明:本评估是规划性文档,未改动代码。任何 P0/P1 的实现都需在能跑
> GUI 的本机用真实台站验证。
