## 1. 决策摘要

| 项 | 决定 |
|---|---|
| 结构 | **双模式**:工作台顶层分「实时监测」/「历史分析」两个自包含模式 |
| 滤波器 | 历史分析用**自己独立的滤波/波段**,与实时分开(不再共用) |
| 删功能 | **不删**:声音(声学化)和 24h helicorder 都保留 |
| 台站"添加到工作台" | **弹"监测实时 / 看历史"二选一**,按选择分流 |
| 事件复核 | **一律进历史模式**,台站落到历史选择器,**不再碰实时 combo** |

---

## 2. 双模式架构

* **🔴 实时监测**:侧边栏 = 纯实时控制台(台站 combo · 连接/停止 · 带通滤波器 ·
  STA/LTA · 声音);内容 = 子页 实时波形 / 24h 记录 / 质点轨迹。三者都是
  "同一连接台站的实时视图"。
* **📊 历史分析**:完全自包含。自己的台站选择器 + 时窗 + **独立滤波/波段** +
  加载/清除 + 导出;内容 = 波形 / 频谱图(UTC) / 功率谱。无任何实时旋钮。

设计目标:实时与历史**各管各的选中台站**,互不污染;事件复核只指向历史模式。

## 4. "加到工作台"落点逻辑(核心)

### 4.1 当前落点(全部入口已核实,main_window)

| 入口 | 加什么 | 当前落点 | 代码 |
|---|---|---|---|
| 地球·点台站·添加 | 台站 | 实时 combo | `_on_globe_station_clicked` → `add_station` (1121) |
| 地球·点地震·复核 | 事件+台站 | replay **+ 实时 combo** | `_on_globe_quake_clicked`→`_open_event_in_replay` (815) |
| 事件中心·双击复核 | 事件+台站 | replay **+ 实时 combo** | `_on_event_review_requested` (839) |
| 事件中心·☆收藏台站 | 台站 | 仅收藏 | `FavoritesStore.add_station` |
| 我的·use 收藏台站 | 台站 | 实时 combo | `_on_use_favorite_station`→`_on_globe_station_clicked` |
| 我的·双击收藏事件 | 事件+台站 | replay **+ 实时 combo** | `_on_review_favorite_event` (920) |
| 我的·录波 / 复核目录 | 历史数据 | replay | `_on_recording_activated` / `_on_review_catalog` |

**问题**:所有台站都塞进实时 combo,连"事件复核"也顺手污染实时 combo(815 行
是 0.8.0 加的,双模式下要拆掉)。

### 4.2 目标落点(双模式)

| 入口 | 意图 | 落点 |
|---|---|---|
| 地球·添加台站 | **二选一** | 弹"监测实时 / 看历史";监测→实时 combo+实时模式;看历史→历史选择器+历史模式 |
| 我的·use 收藏台站 | **二选一** | 同上 |
| 地球/事件中心/我的·**事件复核** | 历史 | **历史选择器 + 历史模式;不碰实时 combo** |
| 录波 / 复核目录 | 历史 | 历史模式 |
| ☆收藏台站 / 事件 | 仅收藏 | 收藏库(不变) |

### 4.3 共享台站池(避免"加了另一模式看不到")

两个模式**各自有自己的选中台站**,但**共享同一个"可选台站池"**:

```
可选台站池 = 实时 combo 已加台站 ∪ 收藏台站 ∪ 当前事件就近台站
```

* 历史选择器的下拉 = 这个池;实时 combo 也从这个池增补。
* 为监测加的台站,在历史下拉里也找得到(反之亦然)。
* **池子共享,选中各自独立**——这就是"软桥"。

## 5. 实施阶段(每阶段过 compile/ruff/纯测试 + i18n 4 语对齐)

1. **顶层双模式** ✅ 完成:`pro_window` 由"4 子页"改为"模式切换 + 实时模式内含
   3 子页";`show_replay()`/`show_live()` API;`is_live/particle_subtab_visible`
   兼顾模式。
2. **侧边栏收为实时专属** ✅ 完成:`_on_mode_changed` 在历史模式隐藏 `control_panel`。
3. **历史模式自包含** ✅ 完成:`replay_panel` 加自己的台站下拉(`station_combo`,
   不可裸输入)+ 独立滤波(`filt_check`/`filt_low`/`filt_high`);断开
   `filter_changed→replay`;`set_station` 改为"加为选项+仅初始选中"(软桥,不劫持)。
4. **重接落点** ✅ 完成:台站"添加"弹"监测实时/看历史"二选一(`select_history_station`);
   事件复核一律进历史模式、台站落回放下拉,删除对实时 combo 的写入。
5. **共享台站池** ❌ **放弃(决策)**:原计划把"收藏台站 + 实时 combo 全部台站"灌进
   回放下拉。**否决理由**:(a) **与「我的」收藏功能重复**——收藏台站本就在"我的"里
   管理与访问("我的"双击→看历史 即可进历史模式),再镜像进下拉等于两处维护同一份
   数据;(b) **下拉膨胀**——收藏 FIFO 上限 100,会让下拉几十条难用,且回放控制行已很挤;
   (c) Stage 4 后收藏台站**已能到达历史模式**(只多一次点击)。
   **改为**:回放下拉 = **本次会话的"工作集"**(实时默认/切过的实时台站/复核的事件台站/
   看历史选过的),自然增量累积;**收藏的唯一的家仍是「我的」**。
   极轻软桥(把实时 combo 那几个台站预灌进下拉)亦从略——其需要"过滤掉不可回溯的 AM"
   的复杂度,收益边际。若将来做"历史浏览框架",台站/收藏的"源"在框架的源侧栏统一呈现,
   不在此处堆叠。
