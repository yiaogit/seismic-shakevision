# 历史回放板块:功能/使用方式分析 + 重构方案

> 目的:对照主流地震软件,诊断"历史回放"(Pro 子标签 #4)的范式问题,
> 给出分阶段重构方案。配套调研见 `seismic-software-survey.md`。

---

## 1. 现状(我们现在是怎么做的)

历史回放当前是一个**"视频播放器"**模型:

1. 选时窗(起始 UTC + 时长)→ 点「下载」从 IRIS dataselect 取 MiniSEED。
2. 构造 `ReplaySource`,像放视频一样**按 N× 倍速回放**;波形面板显示的是
   内部 ring buffer 的**滚动窗**(`display_window_seconds`),随播放推进。
3. 一条进度条 `QSlider` + 播放/暂停/停止 + 倍速下拉(½×–60×)。
4. 台站已改为只读、跟随左侧选中(上一轮);分析工具(冻结/游标/区间/
   P-S 拾取/物理单位)刚打通,复用 Live 那套。

**问题的核心**:用"媒体播放器"的隐喻来做地震波分析。

---

## 2. 主流软件怎么做历史复核(关键差异)

几乎所有专业/准专业软件做"看历史数据"时,**都不是播放,而是静态铺开 + 缩放
平移浏览 + 在静止波形上拾取/测量**:

| 软件 | 历史数据的交互方式 |
|---|---|
| **SWARM**(和本项目最可比) | 载入后整段显示;鼠标在时间轴上**缩放/拖动**;波形/谱/谱图/质点运动四视图;无"倍速播放" |
| **Snuffler**(Pyrocko) | 海量波形档案**流畅缩放/平移/滚动**;静止波形上拾取 P/S、算走时;交互手感标杆 |
| **ObsPyck** | 自动抓波形+响应元数据;静态三分量上设 P/S、标振幅算震级、导出 QuakeML |
| **scolv**(SeisComP) | 事件驱动:选事件→复核震相→定位定震级;叠加**理论到时** |
| **SAC** | 命令/脚本,整段时间序列上拾取/测量/谱 |

**"倍速播放"在专业工具里基本不存在**。它是教学演示用的可视化糖,分析价值低:

* 看不到事件全貌(P→S→coda 一眼扫不到)。
* 动着的波上拾取 P/S 很别扭(刚需"冻结"才能点,本身就说明范式拧巴)。
* 进度条拖拽 ≈ 时间轴平移,**功能重复**且不如直接拖时间轴精确。

**事件驱动**是另一大共性:从目录/地图选一个地震 → 自动填 origin time + 台站 +
按震中距算时窗 + 叠加**理论 P/S 到时**(TauP 走时)。

---

## 3. 回放板块的具体差距

1. **范式错位**:player 而非 static browser(最根本)。
2. **时间轴是相对秒**(0.1…0.9 s),不是 **UTC 绝对时间**——分析要对到时,必须绝对时间。
3. **无理论到时/走时曲线**(ObsPy 自带 TauP,本项目已依赖 ObsPy,几乎零成本)。
4. **无事件驱动入口**:`prefill_from_event()` 已写好却没人调用,地球上点地震不能直接回放。
5. **波段固定**(BH?),同台常有 HH/LH/EH,专业用户想切(上一轮已提)。
6. **无导出**:看完即走,不能存图/CSV/震相/QuakeML,无法复现或归档。

---

## 4. 重构方案:把"播放器"改成"静态波形浏览器"(SWARM / Snuffler 范式)

> 可行性已确认:波形面板 x 轴的缩放/平移**原生已开启**;分析工具(游标/区间/
> P-S/单位/测量)已就绪,直接作用于静态曲线。改动集中在"数据如何进面板"。

### 阶段一 — 核心范式切换(建议优先,自洽,UX 收益最大)
* 下载完成后,把**整段波形一次性静态铺开**(全时窗显示),不再走滚动窗。
* 时间轴改为 **UTC 绝对时间**刻度。
* 用 pyqtgraph 原生**滚轮缩放 + 拖动平移**浏览;加一个「适配全段 / 重置视图」按钮。
* 进度条 + 倍速播放**降级**为可选的「▶ 动画演示」开关(默认关,保留教学用途),
  或直接移除。
* 复用已有 游标/区间/P-S 拾取/物理单位/测量(静态后这些更好用,不再需要"冻结")。

### 阶段二 — 专业增强
* **事件驱动**:地球点地震 → 接通 `prefill_from_event` → 自动算时窗 + 跳到回放标签。
* **理论到时**:用 ObsPy **TauP**(iasp91/ak135),按 事件-台站 震中距叠加 P/S 虚线。
* **波段下拉**(BH/HH/LH/EH),仍以当前台站为准。

### 阶段三 — 导出 / 复现
* 导出 **PNG**(图)/ **CSV**(道数据)。
* 震相拾取 → 文本或 **QuakeML**(对标 ObsPyck)。

---

## 4b. 实施状态(v0.7.7,三阶段全部完成)

* **阶段一 ✅** 静态铺开 + UTC 时间轴(`_UTCAxisItem`)+ 滚轮缩放/拖动 +
  ⟲ 适配全段;分析工具常开;播放器/进度条/倍速**已彻底移除**。
  `WaveformPanel(static_mode=True)` + `load_static()`。
* **阶段二 ✅** 波段下拉(BH/HH/LH/EH/SH);地球点地震 → 确认 → 打开 Replay
  并按震源时刻设时窗(保留当前台站);**TauP(iasp91)理论 P/S** 后台计算并叠加
  (台站坐标取自 StationXML;全程 try/except 降级)。
* **阶段三 ✅** 导出 **PNG**(截图)/ **CSV**(UTC 时间 + Z/N/E)/ **QuakeML**
  (P/S 拾取,经 ObsPy)。

> 注:沙箱无 PySide6/ObsPy 运行环境,以上经 compile + ruff + 纯函数测试验证;
> GUI 行为(UTC 轴渲染、缩放手感、TauP 叠加、导出对话框)需在本机确认。

## 5. 一句话

回放板块当前用"视频播放器"隐喻,与所有主流复核软件(SWARM/Snuffler/ObsPyck/
scolv/SAC)的"**静态铺开 + 缩放平移 + 静止波形上拾取测量**"范式相反。最高性价比的
重构是**阶段一**:把播放器换成静态波形浏览器,倍速播放降级为可选演示。阶段二/三
(事件驱动 + 理论到时 + 导出)让它真正接近 ObsPyck 那一档。
